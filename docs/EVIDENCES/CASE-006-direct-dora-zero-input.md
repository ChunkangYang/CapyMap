# CASE-006: ドラぷら 直接リンク（ゼロ入力）

日期: 2026-05-30

## 解決した問題
ユーザー要望: 「ドラぷら の點進去就能查訊第三方的金額試算、零手動入力」

## アプローチ
1. **テキスト抽出強化**: navigationInstruction を whitespace-strip し、non-greedy regex で IC名抽出、`方面の/出口（交差点）/右側のランプ` 等のノイズを split で除去。
2. **Places API 反查**: text 抽出で JCT しか得られない場合、MERGE/RAMP step の startLocation/endLocation を `rankBy: DISTANCE + keyword: 入口` で検索し、`首都高速○○入口` を取得 → IC raw名 (e.g. "銀座") に変換。
3. **kind=2 (地名検索) URL**: ドラぷら は IC名 + 駅前/市町村名 を kind=2 で受け付ける。エントリ/エグジットそれぞれ「正しいIC raw名 (あれば) / 出発地・目的地の都市名 (なければ)」を使い、URL 生成。
4. **Geocoder 都市名フォールバック**: 「伊豆シャボテン公園」のようにランドマーク住所の場合、`Geocoder.reverse(coord)` の `locality` コンポーネントで都市名を取得。

## 制約 (未解決)
ドラぷら NEXCO東日本 の IC データベースは、NEXCO管理外の道路（伊豆方面の国道135号、横浜新道等）の IC を含まない。
- 「伊東」を arrivePlaceKana にしても該当ICなし → form 画面戻り。
- 「横浜新道」は NEXCO IC ではないため kind=1/2 共に該当なし。

これは ドラぷら 側のデータ制約であり、URL 形式の工夫では解決不可能。
